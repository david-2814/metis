Next steps:

1. Issue your first gateway key. Run inside any pod in the deployment:

   kubectl -n {{ .Release.Namespace }} exec deploy/{{ include "metis-gateway.fullname" . }} -c gateway -- \
     metis gateway issue-key --keystore {{ .Values.keystore.mountPath }} \
       --name "my-client" --workspace /workspace

   The plaintext token is printed once. Save it.

   IMPORTANT: by default the keystore is mounted from a ConfigMap (`{{ include "metis-gateway.keystoreConfigMapName" . }}`),
   which is read-only inside the pod. To persist issued keys across pod restarts:
     a. Bundle the keys.json externally and mount via `keystore.existingSecret`, OR
     b. Patch the ConfigMap with the updated JSON and roll the pod.

   See "Keystore rotation" in docs/gateway-deployment.md.

2. Sanity-check the gateway over the Service:

   kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "metis-gateway.fullname" . }} {{ .Values.service.port }}:{{ .Values.service.port }}
   # then in a second shell:
   curl http://127.0.0.1:{{ .Values.service.port }}/healthz
